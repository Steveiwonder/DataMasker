Hey @advapiIT, great news — this is now supported!

I've added a new `useList` property to the column configuration. You simply provide an array of values and DataMasker will randomly pick one for each row.

For your exact use case with Product Type IDs:

```json
{
    "name": "ProductTypeId",
    "type": "None",
    "useList": [10, 20, 56, 80],
    "retainNullValues": false
}
```

Each row will be assigned a random value from that list. It works with strings too:

```json
{
    "name": "Status",
    "type": "None",
    "useList": ["Active", "Inactive", "Pending"],
    "retainNullValues": false
}
```

The `retainNullValues` behaviour still applies — if the existing value is `null` and `retainNullValues` is `true` (the default), the row will be left as `null`. Set it to `false` if you want nulls to be replaced with a value from the list too.

The README has been updated with full documentation on this. Let me know if you run into any issues!
